## 项目背景

我们正在构建一个面向简体中文用户的 LaTeX 公式自动纠错 SFT 数据集。

整体数据集规划分为三层：

1. `clean`
高质量、清晰、自然的用户输入

2. `noisy`
模拟真实用户不规范但仍可恢复的输入

3. `hard`
更复杂、更长尾、更高难度的输入

当前你审的是 `noisy_v2`，目标不是最终定稿，而是判断它是否已经比 `noisy_v1` 更贴近真实世界，并且是否已经足够稳定，可以继续扩成更大的 noisy 验证集。

---

## 我们已经针对 v1 做过的修正方向

请特别关注下面这些点在 `v2` 中是否真的改好了：

1. 修 `ascii_substitute`
- 减少“自然语言 + 整段 ASCII 片段”的硬拼接
- 改成更自然的局部替换或较短尾部片段

2. 修 `mixed_language`
- 去掉固定的“，就是 xxx”模板
- 改成更自然的中英混输

3. 修 `keyword_shorthand`
- 删除或弱化高歧义简称
- 给某些模糊公式补最小上下文

4. 修 `broken_brackets`
- 不再只是像字符串截断
- 更接近真实输入中的局部结构损坏

5. 轻修 `code_fragment_noise`
- 减少太脚本化的前缀
- 保留代码感，但让它更像真人留下的片段

## 你的审查重点

请不要泛泛而谈，请重点回答下面这些问题：

### 1. v2 相比 v1 是否真的更自然

请明确判断：
- `noisy_v2` 是否减少了“脚本拼接味”
- 是否更像真实用户会打出来的 noisy 输入
- 是否还有明显的模板痕迹残留

### 2. v2 的可恢复性是否更合理

请判断：
- 当前 noisy 输入是否仍能较稳定地恢复到目标 LaTeX
- 是否还有明显的一对多歧义样本
- 哪些规则仍然会制造不可恢复输入

### 3. 哪些问题已经改善了

请具体指出：
- `v1` 中最糟糕的问题里，哪些在 `v2` 里已经明显缓解
- 最好直接按问题类型说，例如：
  - ASCII 拼接
  - 中英混输模板
  - 歧义简称
  - 括号破坏方式

### 4. 哪些问题还没解决

请明确指出：
- `v2` 里仍然残留的高优先级问题
- 哪些 noise_rule 还需要继续重写
- 哪些 noise_rule 只是轻微问题，可以先保留

### 5. 是否可以继续扩量

请给出明确结论：
- 仍然不建议扩量
- 可以小幅修正后扩量
- 已经可以扩成更大的 noisy 验证集
